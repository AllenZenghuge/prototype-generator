### 3.2 特殊控件说明

#### 合同号复合控件 (`.input-with-btns`)
```

```
- 使用 flex 布局，输入框自适应，按钮不缩放
- 按钮使用 `.btn-sm` 小尺寸（24px 高）

#### 单选框组 (`.radio-group`)
- 水平排列，选项间距 16px
- 使用浏览器原生 radio，accent-color 为主色 #1677ff
- 出现位置：合作伙伴类型（客户/供应商）、已预提费用（是/否）、预付核销（是/否）

#### 只读计算字段
- 不含税金额、税额、实际金额：灰色背景 `var(--hx-neutral-3)`，右对齐
- 使用 `readonly` 属性 + 内联样式 `background:var(--hx-neutral-3)`

## 5. 框架集成说明

### 5.1 外壳注入机制（framework.js）

页面本身**不包含**侧边栏、顶栏 HTML。这些由 `framework.js` 在 `DOMContentLoaded` 时注入：

```
页面加载顺序:
1. 浏览器解析 HTML → body 内容（只有页面自身区块）
2. DOMContentLoaded 触发
3. framework.js injectFramework():
   a. 保存 body.innerHTML（页面内容）
   b. 替换 body.innerHTML 为完整外壳 HTML
      (侧边栏 + 顶栏 + .content-inner 容器)
   c. 将保存的页面内容注入 #content-root
   d. 调用 APP.renderSidebar() 渲染菜单
```

**页面检测**：`getPageName()` 根据 `window.location.pathname` 判断：
- 包含 `settlement-form` → 结算单页面配置
- 包含 `invoice-list` → 销项开票页面配置

**结算单页面专用头部**（`buildHeader('settlement-form')`）：
- 主标签：`应付管理 ┆ ⊞ ┆ ⌂ ┆ 结算单-通用(active)`
- 右侧图标：电子税局... ┆ 🔔 ┆ ⚙ ┆ 管
- **无**二级状态筛选栏（草稿/流程中/所有单据）

### 5.3 侧边栏菜单结构（common.js → settlementNav）

页面加载时，`renderSidebar()` 检测到 `settlement-form` 路径，使用 `settlementNav` 数据：

```
合同台账管理
采购结算
系统配置
结算单 ▾ (展开，因含 active 子项)
  ├── 结算单
  ├── 结算单-通用       ← 高亮 (href=settlement-form.html)
  ├── 结算单-八大类
  ├── 结算单-生产采购
  ├── 生产采购台账
  ├── 八大类台账
  ├── 开票通知单执行...
  └── 结算单-权益直兑
在建工程
付款配置
应付票据
应付付款
非生产资金计划
```

**动态激活逻辑** (`applyActive`):
1. 遍历所有菜单项，先全部重置 `active=false`, `open=false`
2. 找到 `href` 匹配 `settlement-form` 的项 → 标记 `active=true`
3. 递归向上标记所有祖先 `open=true`
4. `buildMenuItem` 根据 `open`/`active` 状态添加 CSS 类
